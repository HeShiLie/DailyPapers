## 1-`2604.19748`-`Tstars-Tryon 1.0: Robust and Realistic Virtual Try-On for Diverse Fashion Items`

1. source link: https://arxiv.org/abs/2604.19748
2. 单位：alibaba-inc
3. Keywords：virtual try-on, image generation, fashion editing, commercial deployment
4. 问题背景：虚拟试穿方法在实验室场景里已能生成不错结果，但真实电商环境里会遇到极端姿态、光照变化、模糊和多品类搭配等复杂情况，稳定性和落地效率都不够。工业级系统还必须兼顾细节保真、身份背景控制和实时响应。
5. 动机/Insight：这篇工作的核心 insight 是：商业可用的 VTON 不是只靠一个更强生成模型，而是要把模型结构、数据引擎、训练流程和推理优化作为一体化系统来设计。作者把“高成功率 + 细节真实 + 多参考图可控 + 低延迟”同时当作目标，而不是只卷单一视觉指标。
6. 解决方案：Tstars-Tryon 1.0 构建了覆盖 8 类时尚单品、支持最多 6 张参考图的商业级试穿系统，并采用多阶段训练与系统级优化来提升鲁棒性、写真感和速度。最终不仅给出 benchmark，还已在淘宝 App 大规模部署，面向数百万用户和千万级请求。

## 2-`2604.19636`-`CoInteract: Physically-Consistent Human-Object Interaction Video Synthesis via Spatially-Structured Co-Generation`

1. source link: https://arxiv.org/abs/2604.19636
2. 单位：alibaba-inc
3. Keywords：HOI video synthesis, Diffusion Transformer, Mixture-of-Experts, physical consistency
4. 问题背景：人-物交互视频生成看起来很直观，但现有扩散模型常在手、脸等敏感区域失稳，且容易出现穿模、接触关系错误等物理不一致问题。对电商展示、数字营销这类应用来说，结构稳定和交互真实感是比单帧观感更难的瓶颈。
5. 动机/Insight：作者的关键想法是同时处理两类问题：一类是局部人体区域的精细结构，另一类是人与物之间的交互几何。为此，他们把区域专家路由和结构流共生成结合起来，让模型既学会哪里该重点修，又学会交互关系本身。
6. 解决方案：CoInteract 基于 DiT 提出 Human-Aware MoE 和 Spatially-Structured Co-Generation：前者用空间监督路由把 token 分配给轻量区域专家，后者在训练时并行建模 RGB 外观流与 HOI 结构流，将交互先验注入骨干。推理时结构分支可移除，因此不会额外增加生成开销。

2. 单位：UIUC ScaleML Lab
3. Keywords：LLM agents, DSL, workflow control, state management
4. 问题背景：很多 agent 系统仍主要依赖 prompt 驱动的开放式 ReAct 流程，控制流、状态和中间决策都隐含在自然语言里，导致行为难调试、难复用，也不利于治理与验证。随着 agent 复杂度增长，这种“全靠 prompt 约定”的方式越来越不稳。

6. 解决方案：AgentSPEX 提出一种面向 agent 的领域专用语言与执行框架，用结构化方式定义模块、状态和流程控制。它强调模块化、可解释和可执行，试图把 agent workflow 从隐式 prompt 工程转向显式的软件工程。

1. source link: https://arxiv.org/abs/2604.19295
2. 单位：未在HF页明确标注
3. Keywords：test-time training, reasoning models, critic recalibration, diversity collapse
4. 问题背景：测试时训练希望让模型在推理阶段继续适应无标签样本，但现有方法往往提升一阵就饱和，继续加 test-time compute 也没有线性收益。更糟的是，随着策略模型自我更新，奖励信号会漂移，进而导致探索退化和多样性塌缩。
5. 动机/Insight：作者认为问题不只是 policy 优化不够，而是 policy 和 reward/critic 没有同步校准，导致自我训练很快跑偏。于是他们把“策略改进”和“评价器重标定”做成交替迭代过程。
6. 解决方案：TEMPO 提出一种 test-time training 框架，在推理期间交替进行 policy refinement 与 critic recalibration，以维持持续收益并抑制 diversity collapse。核心贡献是让 TTT 不再只在初期有效，而能更稳定地受益于额外测试时算力。

## 7-`2604.19254`-`ShadowPEFT: Shadow Network for Parameter-Efficient Fine-Tuning`

1. source link: https://arxiv.org/abs/2604.19254
2. 单位：The Hong Kong Polytechnic University
3. Keywords：PEFT, LoRA, layer refinement, shared shadow module
4. 问题背景：PEFT 通过只训练小量参数来替代全参微调，但像 LoRA 这类方法本质上是在各层独立插入局部低秩扰动，适配方式较分散。随着任务更复杂，这种局部参数化未必是最优的。
5. 动机/Insight：这篇文章的核心观点是把 PEFT 从‘按权重打补丁’转成‘按层级做共享精炼’。如果能在层空间里引入一个跨层复用的 shadow state，可能比逐层独立低秩修正更高效。
6. 解决方案：ShadowPEFT 设计了 depth-shared shadow module，在每个 transformer 层并行维护 shadow state，并反复演化以形成更丰富的隐藏表示。它把适配重心从离散的 weight perturbation 转移到集中式 layer-level refinement。

2. 单位：University of California, Berkeley
3. Keywords：video generation, speculative decoding, acceleration, quality-speed tradeoff
4. 问题背景：自回归视频扩散生成推理很慢，现有加速手段主要依赖蒸馏，但 speculative decoding 这种在 LLM 中很成功的思路能否迁移到视频上并不明显，因为视频块是连续时空张量而非离散 token。如何在不显著降质的前提下加速，是核心难点。
5. 动机/Insight：作者的 insight 是用质量导向的 routing 机制代替 token-level exact rejection sampling，让候选块按质量阈值进入目标模型 KV cache。这样 speculative decoding 可被改造为适合视频的近似接受-拒绝流程。
6. 解决方案：该方法将 speculative decoding 引入 autoregressive video generation，通过固定阈值控制质量-速度 Pareto frontier，并配合首块强制拒绝等设计稳定场景构图。结果是在保持接近 target-only 质量的同时获得明显推理加速。
