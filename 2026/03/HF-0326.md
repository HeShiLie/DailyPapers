## 1-`2603.24440`-`CUA-Suite: Massive Human-annotated Video Demonstrations for Computer-Use Agents`

1. source link: https://arxiv.org/abs/2603.24440  
2. 单位：ServiceNow  
3. Keywords：computer-use agents, expert demonstrations, desktop automation, grounding benchmark  
4. 问题背景：Computer-use agent 想真正自动化复杂桌面任务，核心瓶颈之一是缺少**连续、高质量、带过程信息**的人类演示数据。过去不少数据只保留稀疏截图或点击坐标，丢掉了操作时序、鼠标运动和界面动态变化这些关键信号。  
5. 动机/Insight：作者认为，对 CUA 来说，真正稀缺的不是“最终动作标签”，而是**连续视频级的人类操作过程**。只要把完整交互流记录下来，就能向下兼容 grounding、planning、reward modeling、world model 等多种任务。  
6. 解决方案：提出 CUA-Suite 这一整套数据与评测生态，核心包含 VideoCUA（约 1 万任务、87 个应用、约 55 小时连续录屏）、UI-Vision benchmark，以及 GroundCUA grounding 数据。作者还用初步实验说明，现有 foundation action model 在专业桌面场景里仍有明显失败率，说明这类资源确实有必要。

## 2-`2603.24472`-`Why Does Self-Distillation (Sometimes) Degrade the Reasoning Capability of LLMs?`

1. source link: https://arxiv.org/abs/2603.24472  
2. 单位：Microsoft Research  
3. Keywords：self-distillation, reasoning degradation, epistemic verbalization, OOD reasoning  
4. 问题背景：Self-distillation 常被认为能压缩推理链、提升效率，但在数学推理场景里，它并不总是带来收益。尤其是当任务分布变化时，模型可能会出现“更短但更差”的 reasoning。  
5. 动机/Insight：作者把问题归因到 **epistemic verbalization（不确定性表达）被压制**。模型如果在蒸馏中学会“过度自信地快速回答”，在 in-domain 也许更干脆，但在 OOD 问题上会丧失边想边修正的能力。  
6. 解决方案：论文通过控制 teacher 条件信息丰富度与任务覆盖范围，系统分析 self-distillation 对 reasoning 的影响，并在多个模型上观察到明显性能下降。核心贡献不在于提出新训练法，而是揭示：优化 reasoning 不能只看“正确答案轨迹”，还要保留合适程度的不确定性表达。

## 3-`2603.24533`-`UI-Voyager: A Self-Evolving GUI Agent Learning via Failed Experience`

1. source link: https://arxiv.org/abs/2603.24533  
2. 单位：Tencent  
3. Keywords：GUI agent, self-evolving learning, failed trajectory, self-distillation  
4. 问题背景：移动端 GUI agent 往往面对长链路任务、稀疏奖励和失败轨迹利用率低的问题。很多方法要么难以从失败中提炼有效经验，要么 credit assignment 很模糊。  
5. 动机/Insight：作者认为，失败轨迹并不是噪声，而是 GUI agent 自我进化的重要学习材料。关键在于如何把失败经历转成更稳定、更高效的 agent 提升信号。  
6. 解决方案：提出两阶段 self-evolving 框架 UI-Voyager：先通过 rejection fine-tuning 过滤和吸收更高质量经验，再结合 group-relative self-distillation 做进一步优化。整体目标是让 agent 能更高效地从失败中学习，并提升 GUI 自动化性能与样本效率。

## 6-`2603.21289`-`When Models Judge Themselves: Unsupervised Self-Evolution for Multimodal Reasoning`

1. source link: https://arxiv.org/abs/2603.21289  
2. 单位：OPPO  
3. Keywords：multimodal reasoning, unsupervised learning, self-consistency, GRPO  
4. 问题背景：多模态 reasoning 的提升通常依赖人工标注答案、teacher distillation 或外部 reward model，这些都贵而且不易规模化。若想让模型自我进化，就必须摆脱这些高成本监督。  
5. 动机/Insight：作者的核心想法是让模型“自己评自己”，并用**self-consistency + group-relative 比较**替代人工监督。关键不只是打分，而是把组内不同 trajectory 的相对质量转成更稳定的训练信号。  
6. 解决方案：对每个输入采样多条 reasoning trajectory，以 actor 的 self-consistency 为先验，再引入 bounded judge modulation 重加权不同轨迹质量，并转为组内相对优势，最后用 GRPO 训练。这样无需标注答案和外部奖励，也能在多个数学推理 benchmark 上持续提升表现。

## 12-`2603.24036`-`SpectralSplats: Robust Differentiable Tracking via Spectral Moment Supervision`

1. source link: https://arxiv.org/abs/2603.24036  
2. 单位：HF 页面未明确展示（可能为多机构合作）  
3. Keywords：3D Gaussian Splatting, tracking, spectral supervision, frequency domain  
4. 问题背景：把 3D Gaussian Splatting 用到 differentiable tracking 时，一个典型难点是 severe misalignment 下的**vanishing gradient**。一旦渲染对象和目标局部区域几乎不重叠，传统 photometric loss 基本给不出有效梯度。  
5. 动机/Insight：作者意识到，问题并不只是 loss 不够强，而是优化目标定义在空间域时天然依赖 overlap。把目标迁移到频域，能在重叠极差时仍保留可用的优化信号。  
6. 解决方案：提出 SpectralSplats，把监督从空间域改到 frequency domain，用 spectral moments 作为优化目标，并配合 frequency annealing schedule 提高鲁棒性。这样可以缓解 3DGS tracking 中因局部支撑过紧而导致的优化卡死问题。

## 19-`2603.23404`-`Unleashing Spatial Reasoning in Multimodal Large Language Models via Textual Representation Guided Reasoning`

1. source link: https://arxiv.org/abs/2603.23404  
2. 单位：Tsinghua University  
3. Keywords：multimodal LLM, spatial reasoning, textual representation, intermediate reasoning traces  
4. 问题背景：MLLM 在 3D 空间推理上常常不稳定，因为视觉特征虽然丰富，但不一定自然对应明确、可操纵的结构化 reasoning 过程。直接在视觉表征上推理，容易缺少清晰中间变量。  
5. 动机/Insight：作者认为，可以先把视频环境转成**文本化的空间表征**，再在文本空间里做 reasoning。这样能把空间结构显式化，让模型的中间思路更可控、更接近语言模型擅长的推理形式。  
6. 解决方案：提出 TRACE prompting，让模型先生成 text-based representation 作为中间 reasoning trace，再完成 3D spatial reasoning。核心不是改模型参数，而是通过 representation-guided prompting 激发 MLLM 的空间理解潜力。

## 20-`2603.24506`-`Toward Physically Consistent Driving Video World Models under Challenging Trajectories`

1. source link: https://arxiv.org/abs/2603.24506  
2. 单位：HF 页面未明确展示（可能为多机构合作）  
3. Keywords：driving world model, physical consistency, challenging trajectories, simulation  
4. 问题背景：现有 driving video world model 多基于真实驾驶数据训练，因此更擅长自然、安全轨迹；一旦条件换成 simulator 或 planner 产生的反事实、极端或 imperfect trajectory，视频往往会出现明显的物理不一致。  
5. 动机/Insight：作者想解决的不是一般“画面质量”问题，而是 challenging trajectory 下的**physical consistency**。如果 world model 只能拟合常规驾驶分布，那它很难真正服务于仿真和规划。  
6. 解决方案：提出 PhyGenesis，将无效或不合理轨迹变换成更 plausible 的条件，再用 physics-enhanced video generator 在真实与模拟驾驶场景上训练。整体思路是通过显式物理增强，让模型在困难轨迹条件下也能生成更高保真的驾驶视频。

## 21-`2603.23539`-`PLDR-LLMs Reason At Self-Organized Criticality`

1. source link: https://arxiv.org/abs/2603.23539  
2. 单位：Fromthesky Research Labs  
3. Keywords：reasoning, self-organized criticality, phase transition, order parameter  
4. 问题背景：关于 LLM 为什么会表现出 reasoning，很多解释仍停留在经验层面。作者试图从统计物理视角给出一种更“机制化”的解释。  
5. 动机/Insight：论文把 reasoning 看作一种在 self-organized criticality 附近显现的行为，并尝试用 phase transition、correlation length、metastable steady state 等概念去描述。这种视角的重点不是再做 benchmark，而是解释“为什么会推理”。  
6. 解决方案：作者声称，PLDR-LLM 在 criticality 附近的 deductive outputs 呈现类似二阶相变的特征，并可由全局输出参数统计定义 order parameter，用其衡量 reasoning 能力。换言之，论文提出了一套偏理论化的 criticality-based reasoning 解释框架。

## 22-`2603.23478`-`UniFunc3D: Unified Active Spatial-Temporal Grounding for 3D Functionality Segmentation`

1. source link: https://arxiv.org/abs/2603.23478  
2. 单位：HF 页面未明确展示（可能为多机构合作）  
3. Keywords：3D functionality segmentation, active grounding, multimodal LLM, adaptive frame selection  
4. 问题背景：3D scene functionality segmentation 要把自然语言中的交互意图，落到具体可操作部件的精细 mask 上。现有方法通常是碎片化 pipeline，而且在最初任务理解阶段就容易“视觉失明”。  
5. 动机/Insight：作者认为，问题不只是 segmentation 模块不够强，而是系统对时空证据的选择过于被动、启发式。若把 MLLM 当成主动观察者，让它自己决定该看哪些帧、在哪些位置聚焦，grounding 会更统一。  
6. 解决方案：提出训练-free 的 UniFunc3D，把 MLLM 作为 active observer，联合进行 semantic、temporal 和 spatial reasoning，并通过 adaptive frame selection 完成功能分割。它强调 unified active grounding，而不是传统多模块流水线。
