## 1-TAPS-Task Aware Proposal Distributions for Speculative Sampling

  

1. source link: arXiv: https://arxiv.org/abs/2603.27027

2. 单位：Image and Video Understanding Lab

3. Keywords：speculative decoding, draft model, routing, acceptance length

4. 问题背景：Speculative decoding 的加速效果，很依赖 draft model 与下游任务分布是否匹配；但现有 draft 往往训练得比较泛化，任务适配性不清楚。

5. 动机/Insight：作者认为，问题不只是“draft 架构够不够强”，而是“draft 训练数据是否对口”。与其在权重空间粗暴合并多个 draft，不如在推理时按任务特征做路由与组合。

6. 解决方案：训练面向不同数据分布的轻量 draft，并在推理阶段用 confidence-based routing 与 merged-tree verification 进行组合。实验显示，任务专用 draft 在对应 benchmark 上更强，而推理期组合优于简单 checkpoint averaging。

## 2-Towards a Medical AI Scientist

2. 单位：HF 页面未在标题区显式给出统一机构

3. Keywords：medical AI scientist, autonomous research, clinician-engineer co-reasoning, manuscript drafting

4. 问题背景：通用 AI Scientist 虽然能做科研自动化，但在医学场景里，研究必须受医学证据、数据模态差异与伦理约束共同约束，通用框架难直接适用。

## 3-Gen-Searcher-Reinforcing Agentic Search for Image Generation

  

1. source link: arXiv: https://arxiv.org/abs/2603.28767

2. 单位：HF 页面未在标题区显式给出统一机构

3. Keywords：image generation, agentic search, reinforcement learning, search policy

4. 问题背景：图像生成模型常受限于一次性 prompt 或静态生成流程，难像搜索 agent 一样主动探索多步决策路径来提升最终结果。

5. 动机/Insight：作者将图像生成过程视作一种可强化学习的 agentic search，希望模型能通过搜索和反馈逐步找到更优生成轨迹，而不是被动接受单轮输入。

6. 解决方案：提出 Gen-Searcher，把图像生成建模为可强化的搜索过程，通过奖励信号优化 agent 的搜索策略。方法重点在于让模型学会“怎么找更好的生成路径”，而不是只靠更大的生成 backbone。

## 6-HISA-Efficient Hierarchical Indexing for Fine-Grained Sparse Attention

  

1. source link: arXiv: https://arxiv.org/abs/2603.28458

2. 单位：Peking University

3. Keywords：sparse attention, hierarchical indexing, efficiency, long context

4. 问题背景：细粒度 sparse attention 的瓶颈之一在于索引器本身成本不低，序列一长就很容易把效率优势吃掉。

5. 动机/Insight：作者认为，真正要把 sparse attention 做到可扩展，不能只优化注意力计算本身，还得改掉传统 indexer 的二次复杂度负担。

6. 解决方案：提出分层索引机制 HISA，用 hierarchical indexing 替代传统 indexer，把复杂度从 $O(L^2)$ 压到次二次级，同时尽量保持 token 选择质量。方法目标是在长上下文场景下更稳地兼顾效率与精度。

## 7-On Token's Dilemma-Dynamic MoE with Drift-Aware Token Assignment for Continual Learning of Large Vision Language Models

  

1. source link: arXiv: https://arxiv.org/abs/2603.27481

2. 单位：HF 页面未在标题区显式给出统一机构

3. Keywords：continual learning, LVLM, mixture of experts, routing drift

4. 问题背景：大视觉语言模型在持续学习中容易遗忘旧知识，而 MoE 虽然能通过新增专家扩展能力，却会因为 routing drift 带来新旧知识冲突。

5. 动机/Insight：作者抓住的问题是“路由漂移”——不是简单参数覆盖，而是 token 被分错专家，导致旧能力退化。

6. 解决方案：提出 LLaVA-DyMoE，通过动态扩展专家、token 级分配引导和路由正则来缓解持续学习中的遗忘。核心是让新增专家吸收新知识时，不破坏既有专家与 token 的匹配结构。
