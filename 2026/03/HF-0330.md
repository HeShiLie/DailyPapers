## 1-`2603.25716`-`Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models`

  

1. source link: `https://arxiv.org/abs/2603.25716`

2. 单位：H-EmbodVis

3. Keywords：video world models, hybrid memory, occlusion, dynamic subject consistency

4. 问题背景：视频世界模型在处理“主体暂时离开视野后又重新出现”的场景时，常把环境当作静态画布，导致被遮挡/离场主体在回归时出现冻结、变形或直接消失。

5. 动机/Insight：作者认为视频记忆不该只做“背景存档”，还要显式兼顾对动态主体的持续追踪；也就是说，静态背景和运动主体需要两种不同的记忆逻辑。

6. 解决方案：提出 Hybrid Memory 范式，并构建专门面向该问题的 HM-World 数据集。模型侧给出 HyDRA，用 token 化记忆 + 时空相关检索机制，在长时遮挡或出画后尽量保持主体身份与运动连续性。

  

## 2-`2603.25746`-`ShotStream: Streaming Multi-Shot Video Generation for Interactive Storytelling`

  

1. source link: `https://arxiv.org/abs/2603.25746`

2. 单位：未在 HF 页面显式标注

3. Keywords：multi-shot video generation, interactive storytelling, causal architecture, streaming prompts

4. 问题背景：多镜头视频生成对互动叙事很关键，但现有双向架构延迟高、交互差，不适合“边生成边改剧情”的使用方式。

5. 动机/Insight：作者把问题改写成“基于历史上下文做 next-shot generation”，从而把多镜头生成从离线整体规划改成在线流式续写。

6. 解决方案：提出因果式的 ShotStream 架构，并结合双缓存记忆与两阶段蒸馏来兼顾低延迟和跨镜头一致性。这样用户可以在故事进行中持续插入新指令，系统再实时生成后续镜头。

  

## 3-`2603.25730`-`PackForcing: Short Video Training Suffices for Long Video Sampling and Long Context Inference`

  

1. source link: `https://arxiv.org/abs/2603.25730`

2. 单位：Shanda AI Research Tokyo

3. Keywords：long video generation, autoregressive video diffusion, KV-cache, temporal consistency

4. 问题背景：自回归式视频扩散模型做长视频时，KV-cache 线性膨胀、时间重复和误差累积都会迅速恶化生成质量与推理成本。

5. 动机/Insight：作者认为没必要把全部历史一视同仁地保留；长视频生成的关键是把历史上下文做结构化压缩，而不是无节制堆缓存。

6. 解决方案：提出 PackForcing，用三分区的 KV-cache 管理策略配合时空压缩机制，高效维护生成历史。目标是在只做短视频训练的前提下，也能支持更长的视频采样和更长上下文推理。

3. Keywords：LLM agents, skill generation, execution traces, transferable skills

4. 问题背景：给 LLM agent 配技能很重要，但手工写技能扩展性太差；而自动技能生成又容易只学到局部、脆弱、不可迁移的经验。

## 9-`2603.22782`-`Know3D: Prompting 3D Generation with Knowledge from Vision-Language Models`

  

1. source link: `https://arxiv.org/abs/2603.22782`

2. 单位：Peking University

3. Keywords：3D generation, VLM, hidden-state injection, controllable back-view synthesis

4. 问题背景：单视图 3D 生成常因不可见区域缺少强先验而在背面/遮挡区产生随机、不可控甚至不合理的几何结果。

5. 动机/Insight：作者希望把 VLM 的语义理解能力真正接入 3D 生成过程，让“不可见部分的补全”不再只是随机 hallucination，而是能被语言约束。

6. 解决方案：提出 Know3D，通过 latent hidden-state injection 将多模态大模型知识注入 3D 生成流程，并用扩散模型充当语义到几何之间的桥梁。这样可以把 back-view synthesis 变成更可控的语义驱动过程。

## 11-`2603.23906`-`GenMask: Adapting DiT for Segmentation via Direct Mask`

  

1. source link: `https://arxiv.org/abs/2603.23906`

2. 单位：未在 HF 页面显式标注

3. Keywords：segmentation, DiT, direct mask generation, timestep sampling

4. 问题背景：很多分割方法只是把生成模型当 feature extractor 间接适配下游任务，这种路线存在表示失配，也增加了复杂的特征提取流程。

5. 动机/Insight：作者主张把 segmentation 直接写成 generative task；关键难点在于二值 mask 的 VAE latent 分布和自然图像 latent 差别很大，不能直接照搬图像生成设置。

6. 解决方案：提出 GenMask，让 DiT 直接生成黑白分割 mask，并设计偏向极端噪声水平的 timestep sampling 策略，使图像生成与 mask 生成能在同一框架下联合训练。这样避免了为分割专门搭复杂 feature pipeline。

  

## 12-`2603.26664`-`Learning to Commit: Generating Organic Pull Requests via Online Repository Memory`

  

1. source link: `https://arxiv.org/abs/2603.26664`

2. 单位：未在 HF 页面显式标注

3. Keywords：coding agents, pull requests, repository memory, code organicity

4. 问题背景：很多 coding agent 生成的 PR 虽然表面上“能跑”，但不符合项目内部 API、代码风格和隐含架构约束，因此很难被真实维护者接受。

5. 动机/Insight：作者认为仓库快照只能告诉模型“现在长什么样”，却无法告诉它“这个仓库历来怎么改代码”；真正缺的是项目演化过程中的变更模式记忆。

6. 解决方案：提出 Online Repository Memory。agent 通过按时间顺序回放历史 commit、做 supervised contrastive reflection，并把“预测 diff 与 oracle diff 的差距”沉淀成技能，再用于未来 PR 生成，以提高 organicity。

## 14-`2603.22582`-`Lie to Me: How Faithful Is Chain-of-Thought Reasoning in Reasoning Models?`

  

1. source link: `https://arxiv.org/abs/2603.22582`

2. 单位：未在 HF 页面显式标注

3. Keywords：chain-of-thought, faithfulness, transparency, reasoning models

4. 问题背景：CoT 常被当作推理模型可解释性与安全部署中的“透明接口”，但前提是模型说出来的推理过程真能反映其实际决策依据。

5. 动机/Insight：作者指出，CoT 的 faithful 程度并不是固定属性，而会随模型架构、训练方式和提示类型显著变化，因此不能默认“会写推理 = 推理真实”。

6. 解决方案：论文系统评估开源 reasoning model 的 CoT faithfulness，比较不同模型对“影响因素承认率”的差异。结论上，这个问题更像系统设计问题，而非单一模型规模问题。

  

## 15-`2603.24649`-`MedOpenClaw: Auditable Medical Imaging Agents Reasoning over Uncurated Full Studies`

  

1. source link: `https://arxiv.org/abs/2603.24649`

2. 单位：Technical University of Munich

3. Keywords：medical imaging agents, 3D volumes, tool use, evaluation benchmark

4. 问题背景：现有医疗影像 VLM 评测往往基于人工挑选好的 2D 图像，这与真实临床诊断中“主动浏览完整 3D study、多序列取证”的流程相距很远。

5. 动机/Insight：作者强调，真正可用的医疗 agent 不该只对喂给它的单张图作答，而要能在专业查看器里主动导航、检索证据并形成可审计的决策链路。

6. 解决方案：提出 MedOpenClaw 与 MedFlowBench，让模型像临床人员一样通过标准 viewer 动态访问未裁剪的完整 3D 影像 study。实验重点揭示：一旦放到真实工具环境中，现有 VLM 的空间导航与证据整合能力会明显暴露短板。
