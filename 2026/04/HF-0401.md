## 1-CARLA-Air-CARLA-Air: Fly Drones Inside a CARLA World -- A Unified Infrastructure for Air-Ground Embodied Intelligence

  

1. source link: https://arxiv.org/abs/2603.28032

2. 单位：页面未明确披露（作者含 Tianle Zeng 等）

3. Keywords: embodied intelligence, air-ground simulation, UAV, CARLA

4. 问题背景：空地协同具身智能研究缺少统一仿真底座：驾驶仿真通常不含真实飞行动力学，飞行仿真又缺少高保真地面交通场景。跨平台桥接式共仿真还会引入同步与时空一致性问题。

6. 解决方案：CARLA-Air 在单一 Unreal Engine 进程中统一了 CARLA 与 AirSim 风格能力，支持共享物理时钟、统一渲染与多模态传感器采集。它保留原生 Python API 与 ROS 2 接口，便于已有空地代码直接迁移到统一环境中。

6. 解决方案：FIPO 用折扣的 future-KL 信号来重塑策略优化中的奖励分配，从而延长有效 reasoning chain。摘要显示其在数学问题求解上优于常见 GRPO 风格做法。

  

## 3-LongCat-Next-LongCat-Next: Lexicalizing Modalities as Discrete Tokens

  

1. source link: https://arxiv.org/abs/2603.27538

2. 单位：LongCat / Meituan LongCat Team

3. Keywords: multimodal tokens, discrete tokenization, unified modeling, lexicalization

4. 问题背景：统一多模态模型常受制于不同模态表征不统一、接口割裂以及训练目标不一致等问题。尤其视觉、语音等输入很难像文本一样自然融入同一离散序列。

5. 动机/Insight：这篇工作的动机是把不同模态尽量都“词汇化”为离散 token，让多模态建模更像语言建模本身。这样可减少额外适配结构，增强统一性。

6. 解决方案：LongCat-Next 以 lexicalizing modalities as discrete tokens 为核心思路，把多模态内容映射进共享离散 token 空间。按标题和 HF 页面摘要，这是一条偏统一建模范式的路线。
