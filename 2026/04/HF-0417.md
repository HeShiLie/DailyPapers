## 3-RAD-2-RAD-2: Scaling Reinforcement Learning in a Generator-Discriminator Framework

  

1. source link：https://arxiv.org/abs/2604.15308

2. 单位：Huazhong University of Science and Technology

3. Keywords：autonomous driving, reinforcement learning, diffusion policy, generator-discriminator

4. 问题背景：自动驾驶运动规划中，单纯模仿学习难以充分优化长期决策，而直接做强化学习又容易训练不稳。如何在复杂规划场景里同时兼顾生成能力、选择能力与训练稳定性，是一个难点。

5. 动机/Insight：这篇工作的关键点是把规划策略拆成“生成候选轨迹”和“判别选择轨迹”两部分，并把二者放在强化学习闭环里共同优化。生成器负责多样性，判别器负责偏好与筛选，二者形成互补。

6. 解决方案：RAD-2 采用统一的 generator-discriminator 框架：用 diffusion-based generator 生成轨迹，再由 transformer-based discriminator 选择和反馈。配合 imitation initialization、closed-loop rollout 和多阶段优化，提升自动驾驶规划的稳定性与性能。

  

## 4-Teacher-Student Cooperation Framework-How to Fine-Tune a Reasoning Model? A Teacher-Student Cooperation Framework to Synthesize Student-Consistent SFT Data

  

1. source link：https://arxiv.org/abs/2604.14164

2. 单位：页面未明确披露具体单位（论文页列出 9 位作者）

3. Keywords：reasoning model, supervised fine-tuning, synthetic data, teacher-student

4. 问题背景：给 reasoning model 做 SFT 时，直接拿强 teacher 生成的数据并不总是有效，甚至可能导致推理能力下降。根源之一是 teacher 的推理风格和 student 自身分布不一致，形成 off-policy 式失配。

5. 动机/Insight：作者认为，问题不在于 synthetic data 本身，而在于数据“像不像 student 会写出来的东西”。因此应让 teacher 与 student 协作生成更贴近 student 风格的训练样本，而不是单边灌输。

6. 解决方案：该方法构建 teacher-student cooperation 的数据合成框架，专门生成 student-consistent 的 SFT 数据。目标是减小 teacher/student 风格分布偏差，从而让 reasoning model 的微调更稳定、收益更可靠。

  

## 5-GlobalSplat-GlobalSplat: Efficient Feed-Forward 3D Gaussian Splatting via Global Scene Tokens

  

1. source link：https://arxiv.org/abs/2604.15284

2. 单位：The Hebrew University of Jerusalem

3. Keywords：3D Gaussian Splatting, feed-forward reconstruction, scene tokens, efficiency

4. 问题背景：许多 3DGS 方法仍然依赖密集高斯或逐像素分配 primitive，虽然效果好，但存储和推理成本偏高。对实际部署来说，更紧凑、更快的场景表达依然很重要。

5. 动机/Insight：作者不再把高斯数量绑定到像素级密度，而是让模型先学习一组压缩的全局 scene tokens，再从这些全局表示统一解码场景。这样可以先建模全局一致性，再下放到高斯表示。

6. 解决方案：GlobalSplat 是一种 feed-forward 3DGS 方法，用全局 scene tokens 代替逐像素 primitive 分配。它用更少的高斯数（如 2K–32K）实现高质量重建，同时带来更小磁盘占用与更快单次推理。

  

## 6-ASGuard-ASGuard: Activation-Scaling Guard to Mitigate Targeted Jailbreaking Attack

  

1. source link：https://arxiv.org/abs/2509.25843

2. 单位：Korea University

3. Keywords：LLM safety, jailbreak defense, activation intervention, mechanistic safety

4. 问题背景：针对性 jailbreaking 往往能绕过基于数据对齐的安全机制，因为攻击目标是模型内部的特定脆弱路径。仅靠 prompt-level 对齐或拒答规则，防御上常常不够稳。

5. 动机/Insight：作者把防御焦点从“输入怎么改”转向“中间激活怎么控”，认为 targeted jailbreak 的关键脆弱性存在于内部 activation dynamics。既然攻击在模型内部起作用，防御也应直接作用于内部表征。

6. 解决方案：ASGuard 提出 activation-scaling guard，通过干预模型内部激活来缓解 targeted jailbreak。它属于 mechanistic safety 路线，强调对内部计算过程而非仅输入/输出层面的安全控制。

## 12-Re2Pix-Representations Before Pixels: Semantics-Guided Hierarchical Video Prediction

## 13-TRACER-TRACER: Trace-Based Adaptive Cost-Efficient Routing for LLM Classification

  

1. source link：https://arxiv.org/abs/2604.14531

2. 单位：DeepRecall

3. Keywords：LLM routing, classification, surrogate model, cost reduction

4. 问题背景：很多分类任务直接用大模型推理，效果好但代价高，而且并非所有样本都需要昂贵模型处理。问题在于，怎样安全地把简单样本分流到便宜模型上。

5. 动机/Insight：TRACER 利用生产 trace 训练 surrogate model，并只在 surrogate 与原始 LLM 高置信一致时启用替代。这样不是一刀切降级，而是基于边界可解释的自适应路由。

6. 解决方案：TRACER 是一个面向 LLM classification 的开源系统：使用 traces 学习小模型近似器，并设置 agreement threshold 控制何时替代 LLM。目标是在保持可控和可解释的前提下降低调用成本。
