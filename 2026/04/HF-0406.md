## 1-GrandCode-GrandCode: Achieving Grandmaster Level in Competitive Programming via Agentic Reinforcement Learning

  

1. source link: https://arxiv.org/abs/2604.02721

2. 单位：DeepReinforce

3. Keywords：competitive programming, multi-agent RL, agentic GRPO, online test-time RL

4. 问题背景：竞赛编程一直是代码智能体最难真正超越人类顶尖选手的场景之一。以往系统即便在离线评测上表现强，也很难在真实 live contest 的多阶段推理、验证与博弈环境下稳定拿到顶尖成绩。

5. 动机/Insight：作者认为，想把模型推到“竞赛级”水平，关键不只是更强的基础模型，而是把假设提出、求解、测例生成、总结等模块组织成一个协同 agent 系统，并让这些模块在真实竞争信号下共同进化。另一个核心洞察是，传统 RL 在多阶段 agent rollout、延迟奖励和 off-policy drift 下并不稳，需要专门面向 agentic setting 的优化方法。

6. 解决方案：论文提出 GrandCode，一个面向竞赛编程的多智能体强化学习系统，通过模块化 agent 协作完成解题流程，并结合 post-training 与 online test-time RL 持续提升。方法层面引入 Agentic GRPO，以更好处理多阶段 rollout 的奖励归因与策略漂移问题，最终在多场 Codeforces live 比赛中达到超过人类顶尖选手的水平。

  

## 2-InCoder-32B-Thinking-InCoder-32B-Thinking: Industrial Code World Model for Thinking

  

1. source link: https://arxiv.org/abs/2604.03144

2. 单位：页面未明确披露（HF 页面显示 25 authors）

3. Keywords：industrial code reasoning, error-driven CoT, code world model, hardware-aware coding

4. 问题背景：工业软件开发与通用代码生成不同，往往涉及芯片设计、GPU 优化、嵌入式系统等强约束场景，需要显式考虑硬件约束、时序语义和错误定位。现有代码模型通常缺少这类“专家级推理轨迹”，导致在工业任务里 reasoning 质量和可用性不足。

5. 动机/Insight：作者的出发点是，工业代码能力的瓶颈并不只在代码 token 预测，而在于缺乏围绕错误、约束和执行反馈展开的思考链。只要能合成并学习这类贴近工程实践的 reasoning traces，模型就有机会获得更接近工业专家的推理模式。

6. 解决方案：论文提出 InCoder-32B-Thinking，基于 Error-driven Chain-of-Thought（ECoT）合成框架与工业代码 world model（ICWM）构造训练数据。模型通过学习错误驱动的推理链和领域执行轨迹，生成更高质量、更加硬件/约束感知的代码 reasoning 与实现。

2. 单位：页面未明确披露（HF 页面显示 10 authors）

3. Keywords：RLVR, self-distillation, on-policy distillation, verifiable rewards

4. 问题背景：带可验证奖励的强化学习（RLVR）能直接利用环境反馈，但通常只有稀疏的 outcome-level 信号，训练容易不稳定、更新粒度粗。相比之下，蒸馏范式能提供更细粒度监督，但又往往依赖更强教师模型，成本和依赖都较高。

5. 动机/Insight：这篇工作的核心想法是，把 RLVR 的环境真反馈与蒸馏式细粒度更新结合起来，但不再依赖外部大教师，而是做“自蒸馏”。作者认为，这样既能保留可验证奖励带来的正确优化方向，也能获得更稳定、更细致的策略更新。

6. 解决方案：论文提出 RLSD，将 RLVR 与 self-distillation 统一起来，让模型在 on-policy 采样过程中利用自身高质量轨迹提供蒸馏信号。这样一来，训练既受环境可验证反馈约束，又能通过蒸馏改善梯度稳定性和更新密度。

  

## 4-A Simple Baseline for Streaming Video Understanding-A Simple Baseline for Streaming Video Understanding

  

1. source link: https://arxiv.org/abs/2604.02317

2. 单位：LMMs-Lab

3. Keywords：streaming video understanding, sliding window, video LLM, online perception

4. 问题背景：流式视频理解近年越来越依赖复杂记忆机制，以应对长视频流中的历史信息保留和实时响应问题。但这些方法系统复杂、成本较高，也未必真正带来比简单策略更强的在线理解能力。

5. 动机/Insight：作者想检验一个很直接的问题：复杂 memory 机制是否真的必要。论文发现，只输入最近的 N 帧、配合现成 VLM 的 sliding-window 基线，已经能匹配甚至超过不少已发表 streaming 方法，说明“近期感知”本身可能比复杂记忆更重要。

6. 解决方案：论文提出 SimpleStream，将 streaming video understanding 形式化为一个极简的 recent-window 输入策略。作者在 OVO-Bench 和 StreamingBench 上，把它和 13 个 offline/online 视频模型比较，展示了简单基线在实时感知场景中的强竞争力，也揭示了实时感知与长程记忆之间的 trade-off。

6. 解决方案：论文提出 Agentic-MME，一个面向多模态 agent 能力的 benchmark，联合评估 visual tool 与 web search 等能力。它不仅检查最终答案，还验证工具调用的真实性、正确性和效率，从而更贴近真实 multimodal problem solving 的需求。

  

## 6-Token Warping Helps MLLMs Look from Nearby Viewpoints-Token Warping Helps MLLMs Look from Nearby Viewpoints

  

1. source link: https://arxiv.org/abs/2604.02870

2. 单位：页面未明确披露（HF 页面显示 6 authors）

3. Keywords：token warping, MLLM, viewpoint transformation, visual reasoning

4. 问题背景：多模态大模型在视觉推理上看起来很强，但一旦视角发生轻微变化，性能往往明显下降。传统的 pixel-wise warping 对深度误差很敏感，容易引入几何畸变，导致从邻近视角“看世界”的建模并不稳。

5. 动机/Insight：作者提出，既然 MLLM 最终消费的是视觉 token，那么与其在像素空间硬做 warping，不如直接在 token 层建模视角变化。这样有望减少像素几何误差传播，同时保持更稳定的语义一致性。

6. 解决方案：论文探索 token-level warping 来帮助 MLLM 从附近视角理解场景，并将其与传统 pixel-wise 方案对比。结果显示，这种做法在稳定性、语义连贯性和相关视觉推理任务上都更有优势。

  

## 7-Test-Time Scaling Makes Overtraining Compute-Optimal-Test-Time Scaling Makes Overtraining Compute-Optimal

  

1. source link: https://arxiv.org/abs/2604.01411

2. 单位：University of Wisconsin-Madison

3. Keywords：scaling laws, test-time scaling, overtraining, compute-optimality

4. 问题背景：经典预训练 scaling law 主要考虑模型大小与训练 token 的最优配比，但现代 LLM 往往会在测试时通过 repeated sampling 等手段做 test-time scaling。这样一来，推理阶段的成本不再可忽略，传统的最优训练结论未必还成立。

5. 动机/Insight：作者的关键观点是，训练与推理预算应该一起考虑，而不是分开讨论。把 inference sample 数也纳入总算力预算后，原本看似“过训”的策略可能反而成为全流程最优。

6. 解决方案：论文提出 Train-to-Test（T^2）scaling laws，在固定端到端预算下联合优化模型规模、训练 token 数和测试采样数。分析表明，当 test-time scaling 被纳入目标后，最优预训练决策会偏向传统意义上的 overtraining 区域。

## 10-Swift-SVD-Swift-SVD: Theoretical Optimality Meets Practical Efficiency in Low-Rank LLM Compression

  

1. source link: https://arxiv.org/abs/2604.01609

2. 单位：Tele-AI

3. Keywords：LLM compression, low-rank approximation, SVD, KV cache reduction

4. 问题背景：大语言模型部署会同时受到静态权重和动态 KV cache 的内存/带宽开销制约。SVD 类低秩压缩方法硬件友好、实现直接，但已有方法要么重建误差不够优，要么理论上最优却工程上代价太高。

5. 动机/Insight：作者希望同时拿到“理论最优性”和“实际可用性”，而不是在两者间二选一。核心洞察是，把协方差聚合和特征分解做得更高效，就有机会在不牺牲最优性的前提下降低压缩代价。

6. 解决方案：论文提出 Swift-SVD，一个面向 LLM 低秩压缩的框架，通过高效的 covariance aggregation 与 eigenvalue decomposition 获得最优低秩近似。该方法旨在在压缩质量、计算效率与实际部署可行性之间取得更优平衡。

## 12-VLMs Need Words-VLMs Need Words: Vision Language Models Ignore Visual Detail In Favor of Semantic Anchors

  

1. source link: https://arxiv.org/abs/2604.02486

2. 单位：页面未明确披露（HF 页面显示 7 authors）

3. Keywords：VLMs, fine-grained perception, semantic anchors, naming effect

4. 问题背景：VLM 在许多多模态任务上成绩很好，但在需要精细视觉辨别的任务上常常掉链子。奇怪的是，这些失败并不总是因为模型表征里没有相关信息，而是它没有把这些视觉细节有效转成最终决策。

5. 动机/Insight：作者指出，问题根源在于 VLM 的训练范式过度偏向语言空间：模型倾向先把视觉对象映射到文本标签，再以“语言方式”完成任务。于是，有名字的目标更容易被处理，而无名或新奇视觉实体即使被看到了，也更难被正确利用。

6. 解决方案：论文通过自然图像、抽象形状和人脸上的 correspondence 任务分析这一现象，并发现一旦给新形状教一个任意名字，性能就会显著提升。由此，工作系统揭示了 VLM 依赖 semantic anchors、忽略细粒度视觉细节的机制。

  

## 13-AgentSocialBench-AgentSocialBench: Evaluating Privacy Risks in Human-Centered Agentic Social Networks

  

1. source link: https://arxiv.org/abs/2604.01487

2. 单位：Carnegie Mellon University

3. Keywords：privacy, agentic social networks, multi-agent coordination, abstraction paradox

4. 问题背景：随着以人为中心的 agent 社交网络逐渐成为现实，多个协作 agent 会跨域、跨用户交换信息，带来前所未有的隐私泄露风险。已有研究虽然讨论过多智能体协作和单 agent 隐私，但还没有系统刻画这种“社交网络式 agent 生态”里的隐私问题。

5. 动机/Insight：作者的一个重要发现是，多 agent 协作本身会持续制造泄露压力，哪怕系统明确要求保护隐私也不够。更反直觉的是，教 agent 用更抽象方式表达敏感信息，反而可能让它更频繁地讨论这些信息，即所谓 abstraction paradox。

6. 解决方案：论文提出 AgentSocialBench，这是首个系统评估 human-centered agentic social networks 隐私风险的 benchmark。它构建覆盖七类场景的 realistic user profiles、敏感度层级和社交图结构，用来测量跨域、跨人协作中的隐私泄漏模式。

5. 动机/Insight：作者认为，视觉表征不该只押注单一路线，而应该把对齐能力强的 contrastive encoder 与语义更丰富的 self-supervised encoder 互补起来。关键在于设计合适的融合机制，让两类 encoder 真正形成互补，而不是简单拼接。

6. 解决方案：论文提出 CoME-VL，通过 entropy-guided aggregation 和 RoPE-enhanced attention 融合两类视觉编码器。结果表明，这种 complementary multi-encoder 设计能够提升视觉理解与 grounding 等任务表现。

## 16-Salt-Salt: Self-Consistent Distribution Matching with Cache-Aware Training for Fast Video Generation

  

1. source link: https://arxiv.org/abs/2604.03118

2. 单位：页面未明确披露（HF 页面显示 9 authors）

3. Keywords：video generation, distribution matching, low NFE, cache-aware training

4. 问题背景：视频生成若想真正走向实时部署，就必须在极低 NFE 下仍保持可接受的画质和运动质量。但传统 trajectory-style consistency distillation 在复杂视频动态下容易变保守，生成结果常常过平滑、运动偏弱。

5. 动机/Insight：作者认为，问题不只是蒸馏不够强，而是现有蒸馏目标没有很好约束不同推理步数/状态下的一致性。若能显式做 self-consistent distribution matching，再结合缓存感知训练，模型在极低推理预算下仍可能保持更强的视频质量。

6. 解决方案：论文提出 Salt，用 self-consistent distribution matching 来蒸馏视频生成模型，并加入 cache-aware training 以适应实时自回归生成。该方法主要面向 2–4 NFE 这类极端低预算场景，目标是在速度和质量之间取得更优折中。

  

## 17-DriveDreamer-Policy-DriveDreamer-Policy: A Geometry-Grounded World-Action Model for Unified Generation and Planning

  

1. source link: https://arxiv.org/abs/2604.01765

2. 单位：GigaAI-Research

3. Keywords：world-action model, autonomous driving, geometry grounding, unified planning

4. 问题背景：世界-动作模型试图统一 world model 与 VLA 的优势，但已有工作往往更偏 2D 外观或 latent 建模，几何约束并不充分。对于自动驾驶这类强物理、强空间任务来说，缺少几何 grounding 会直接限制规划与预测质量。

5. 动机/Insight：作者的核心判断是，真正可用的 driving world-action model 不能只会“看着像”，而必须在表示层就对深度、运动和空间结构有更强几何约束。只有这样，生成未来与做动作规划才能落在同一个可信的世界表示上。

6. 解决方案：论文提出 DriveDreamer-Policy，将深度生成、未来视频预测与运动规划整合进统一的模块化框架。方法通过 geometry-aware world representation 学习，把生成与规划联结起来，在导航类基准上取得更强表现。
