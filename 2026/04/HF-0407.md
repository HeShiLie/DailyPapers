## 1-Adam's Law: Textual Frequency Law on Large Language Models

1. source link: https://arxiv.org/abs/2604.02176

2. 单位：FaceMind

3. Keywords：Textual Frequency Law, prompt optimization, curriculum fine-tuning, data distillation

4. 问题背景：LLM 对语义等价但表达频率不同的文本并不等价：罕见表述更容易让模型理解和学习失真。现有研究很少系统讨论“文本频率”这一潜在影响因素。

5. 动机/Insight：作者提出 Textual Frequency Law，认为高频表达在 prompting 和 fine-tuning 中都更利于 LLM 发挥。也就是说，模型不仅在学知识，也在强烈继承训练语料的表达分布偏好。

6. 解决方案：论文围绕这一规律提出三件事：输入改写为更高频表达的 paraphraser、利用续写构造更高频训练语料的 TFD、以及按频率排序训练样本的 CTFT。实验覆盖数学、翻译、常识推理和 tool calling，整体表明频率建模能稳定带来收益。

2. 单位：Peking University

3. Keywords：world models, unified codebase, perception-action-memory, open-world agents

4. 问题背景：世界模型方向概念繁杂、实现分散，研究者很难在统一框架下比较“感知—交互—记忆—预测”等能力。缺少清晰定义也限制了先进世界模型的系统化发展。

5. 动机/Insight：作者想做的不只是一个代码库，而是给“advanced world models”下统一定义。核心判断是：真正有用的世界模型应同时具备环境理解、可交互性和长时记忆，而不是只会短期视频预测。

6. 解决方案：论文提出 OpenWorldLib 作为统一代码库与任务定义框架，用标准化接口整合感知、决策、记忆与环境建模模块。其目标是让不同世界模型在同一基座上被训练、复现和评测，从而推动开放世界场景下的通用建模。

2. 单位：OpenDataLab

3. Keywords：document parsing, data-centric AI, training data engineering, OmniDocBench

4. 问题背景：文档解析近年多把竞争焦点放在结构设计和模型规模上，但不同模型往往会在同一批困难样本上失误。这个现象说明瓶颈可能并不主要来自架构，而是训练数据覆盖与质量不足。

5. 动机/Insight：作者的核心判断很直接：文档解析的 SOTA 上限，很大程度被数据工程而不是新 backbone 卡住。既然错误模式跨模型一致，就应该回到 hard samples、标注策略和训练分布本身去做优化。

6. 解决方案：论文在不改模型架构的前提下，系统优化训练数据工程与训练策略，推动文档解析性能提升。最终方法在 OmniDocBench v1.6 上取得 SOTA，强调 data-centric 路线本身就能把上限显著往上推。


## 4-TriAttention: Efficient Long Reasoning with Trigonometric KV Compression

1. source link: https://arxiv.org/abs/2604.04921

2. 单位：NVIDIA

3. Keywords：KV cache compression, long-context reasoning, RoPE, efficient attention

4. 问题背景：长上下文推理最大的系统瓶颈之一是 KV cache 占用，而现有压缩方法常用 post-RoPE attention 来估计 key 重要性。问题在于 query 会随位置旋转，代表性 query 很少，导致压缩过程不稳。

5. 动机/Insight：作者抓住了 RoPE 带来的“重要性估计偏差”这一根因，认为应回到更稳定的 pre-RoPE 表征去判断哪些 key 真正值得保留。这样才能在长推理时既压缩内存，又不把关键证据误删。

6. 解决方案：TriAttention 通过三角函数结构建模 Q/K 在 pre-RoPE 空间中的集中性，用更可靠的 key 重要性估计来做 KV 压缩。结果是显著缓解长推理内存压力，同时保持较稳定的生成与 reasoning 性能。

## 6-Memory Intelligence Agent

6. 解决方案：论文提出 Memory Intelligence Agent，把 non-parametric memory、parametric memory 和 RL 训练整合到统一框架中。目标是在开放环境中实现更高效的推理、持续适应与自主演化。

## 8-Can LLMs Learn to Reason Robustly under Noisy Supervision?

1. source link: https://arxiv.org/abs/2604.03993

2. 单位：· 10 authors

3. Keywords：RLVR, noisy supervision, label refinement, mathematical reasoning

4. 问题背景：RLVR 的一个现实问题是标签和奖励并不总是干净，尤其在自动构造监督或大规模采样场景中，噪声会误导策略更新。若缺少抗噪机制，模型甚至会被错误奖励越训越偏。

5. 动机/Insight：作者认为不能把 noisy labels 当作固定污染，而应随着 policy 变强不断重估。也就是说，标签修正本身应成为训练过程的一部分，而不是预处理阶段的一次性动作。

6. 解决方案：论文提出 Online Label Refinement，根据策略提升与一致性信号逐步修正监督标签。该方法在多个数学推理 benchmark 上提升了 RLVR 在噪声环境下的稳健性。

## 12-LightThinker++: From Reasoning Compression to Memory Management

1. source link: https://arxiv.org/abs/2604.03679

2. 单位：Zhejiang University

3. Keywords：reasoning compression, memory management, long-horizon agents, token efficiency

4. 问题背景：长推理链虽然提升了解题能力，但也带来巨大 token 与时间开销。单纯压缩中间思维有时会丢失关键细节，尤其在复杂推理和长时 agent 场景中更容易卡住。

5. 动机/Insight：作者从“静态压缩思维”进一步走向“显式管理记忆”，认为高效 reasoning 不只是缩短 CoT，而是决定哪些信息该被保留、何时调取、何时丢弃。重点从表示压缩上升到了行为层面的 memory scheduling。

6. 解决方案：LightThinker 先把中间推理压成紧凑语义表示，LightThinker++ 则加入显式 memory primitives 和轨迹合成训练管线。实验显示其能大幅减少 token 占用，并在标准推理和长时 agent 任务中同时保持甚至提升性能。


## 13-BidirLM: From Text to Omnimodal Bidirectional Encoders by Adapting and Composing Causal LLMs

1. source link: https://arxiv.org/abs/2604.02045

2. 单位：BidirLM (Decoder To Encoder)

3. Keywords：bidirectional encoder, causal LLM adaptation, omnimodal representation, model composition

4. 问题背景：将 decoder-only LLM 变成强 encoder 很有吸引力，但现有方法在训练目标、规模扩展和多模态整合上都还不稳定。尤其一旦粗暴改造，常会出现灾难性遗忘或能力折损。

6. 解决方案：BidirLM 系统比较不同适配方案，并提出新的 merging / composing 策略，将 causal generative LM 转成 omnimodal bidirectional encoders。结果在多种模态与任务上取得了有竞争力甚至更优的表现。


## 14-Self-Execution Simulation Improves Coding Models

1. source link: https://arxiv.org/abs/2604.03253

2. 单位：· 7 authors

3. Keywords：code LLM, execution simulation, competitive programming, RLVR

4. 问题背景：代码模型常能写出表面像样的程序，但缺乏对执行过程的内化理解，因此在复杂编程题上容易出错。单看最终代码而不模拟运行过程，会让训练信号过于稀疏。

5. 动机/Insight：作者把“会执行程序”视为编码能力的一部分，而非外部 verifier 的附属物。若模型能在生成时显式模拟程序一步步如何运行，它就更可能发现 bug、保持状态一致并改进解题。

6. 解决方案：论文让 code LLM 学习 step-by-step 的自执行模拟，并结合监督微调与可验证奖励 RL 来强化这种能力。最终在 competitive programming 任务上带来明显性能提升。

## 16-Unifying Group-Relative and Self-Distillation Policy Optimization via Sample Routing

1. source link: https://arxiv.org/abs/2604.02288

2. 单位：· 9 authors

3. Keywords：GRPO, SDPO, sample routing, RLVR

4. 问题背景：GRPO 和 SDPO 分别代表两类常见的 post-training 路线：前者强调相对奖励驱动，后者强调自蒸馏式修正。两者各有优点，但通常被当作彼此独立的方法族。

5. 动机/Insight：作者认为关键不是在算法层面二选一，而是根据样本状态去“分流”优化方式。正确样本更适合 reward-aligned 强化，失败样本则更适合做 targeted correction。

6. 解决方案：论文提出 Sample-Routed Policy Optimization，通过 sample routing 将 GRPO 与 SDPO 统一到一个框架里。它把不同样本送往不同更新路径，从而兼顾训练稳定性与最终性能。

## 19-PLUME: Latent Reasoning Based Universal Multimodal Embedding

1. source link: https://arxiv.org/abs/2604.02073

2. 单位：Chinese Academic of Science Institute of Automation

3. Keywords：multimodal embedding, latent reasoning, retrieval, continuous reasoning states

4. 问题背景：通用多模态 embedding 往往难以表达复杂查询意图，于是近来有人在取 embedding 前先显式生成 CoT。问题是这种做法推理开销大，而且会把丰富多模态信息压进狭窄的文本瓶颈。

## 20-Learning to Learn-at-Test-Time: Language Agents with Learnable Adaptation Policies

1. source link: https://arxiv.org/abs/2604.00830

2. 单位：National University of Singapore

3. Keywords：test-time learning, adaptation policy, bi-level optimization, language agents

4. 问题背景：语言 agent 在新环境中常需要 test-time adaptation，但现有策略多依赖人工设计规则，难以跨任务泛化。真正缺的是“如何适应”的学习机制，而不是又一个手工技巧。

5. 动机/Insight：作者把 test-time adaptation policy 本身视为可学习对象。只要把它写成 bi-level optimization，agent 就能学会何时更新、用什么信息更新、以及更新到什么程度。

## 21-Synthetic Sandbox for Training Machine Learning Engineering Agents

1. source link: https://arxiv.org/abs/2604.04872

2. 单位：AI at Meta

3. Keywords：MLE agents, synthetic environments, on-policy RL, sandbox

4. 问题背景：和 SWE agent 相比，MLE agent 的验证成本高得多，因为每条 rollout 都可能涉及完整的数据处理、模型训练和评估流水线。于是 on-policy RL 在该领域长期不现实。

5. 动机/Insight：作者抓住了瓶颈的真正来源：不是任务本身太难，而是 sandbox 数据规模太大、验证太慢。只要构造足够真实但更小规模的可验证环境，MLE agent 也能做 trajectory-wise RL。

6. 解决方案：论文提出 SandMLE，多 agent 生成框架可从少量 seed tasks 合成多样 MLE 环境，并把每个任务的数据集压到 50–200 样本级别。这样把执行时间降了 13 倍以上，使得 MLE domain 的大规模 on-policy RL 首次变得可行。
