## 3-RationalRewards-RationalRewards: Reasoning Rewards Scale Visual Generation Both Training and Test Time

1. source link
   [arXiv:2604.11626](https://arxiv.org/abs/2604.11626)

2. 单位
   · 6 authors

3. Keywords
   Reward Model, Visual Generation, Reasoning Critique, RL

4. 问题背景
   视觉生成 reward model 往往只输出一个标量分数，丢失了人类偏好背后的细粒度判断依据，因此既难解释，也难在训练中提供高质量反馈。

5. 动机/Insight
   如果 reward model 不只是“打分”，而是先给出结构化 critique，再评分，那么它既能服务 RL，也能在 test-time 直接帮助改 prompt。

6. 解决方案
   RationalRewards 训练可生成显式推理 critique 的 reward model，并用其在训练阶段提供细粒度奖励、在测试阶段执行 generate-critique-refine 循环，推动视觉生成质量提升。 ([arXiv][4])

2. 单位
   Princeton University

3. Keywords
   On-policy Distillation, Token Selection, Student Entropy, Teacher-Student Divergence

4. 问题背景
   OPD 默认对所有 token 都做监督，但不同位置的学习价值并不相同，导致训练成本高且很多监督信号并不关键。

5. 动机/Insight

# 当天趋势小结

## 1. 4 月 16 日继续明显偏向 agent / system

GUI agent、coding agent、professional agent、game agent、personal agent 都很多，说明社区兴趣已经很明显地从“单模型能力”转向“可执行系统能力”。([Hugging Face][1])

## 2. OPD / RL / post-training 还是高频主题

TIP、Self-Distillation Zero、Target Policy Optimization、Pre-train Space RL、TREX 都在讨论后训练信号、token 选择或自动化优化流程，说明这一块仍在快速细化。([Hugging Face][8])

## 3. GUI 方向很热，而且开始分化成多个子问题

同一天里既有 GUI grounding（UI-Zoomer），也有 long-horizon GUI automation（UI-Copilot），还有 mobile GUI threat/security 分析，说明 GUI agent 已从单点能力扩展成完整研究簇。([Hugging Face][9])

## 4. 多模态也更强调“真实场景适配”

从区域文化适配、HDR 视频、物理属性重建到噪声网页检索，这些工作都不只是追求更高分，而是在补足模型走向真实世界时的缺口。([arXiv][16])
