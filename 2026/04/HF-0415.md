## 1-ClawGUI-ClawGUI: A Unified Framework for Training, Evaluating, and Deploying GUI Agents

**1. source link**
   [https://arxiv.org/abs/2604.11784](https://arxiv.org/abs/2604.11784)

**4. 问题背景**

## 2-KnowRL-KnowRL: Boosting LLM Reasoning via Reinforcement Learning with Minimal-Sufficient Knowledge Guidance

**1. source link**
   [https://arxiv.org/abs/2604.12627](https://arxiv.org/abs/2604.12627)

**2. 单位**
   Tianjin University

**3. Keywords**
   LLM Reasoning, Reinforcement Learning, Minimal-Sufficient Guidance, Hint Selection

**4. 问题背景**
   RL 提升推理能力时，常因奖励稀疏而训练困难。现有 hint-based 方法通常靠加入更多提示信息缓解稀疏，但也会引入冗余、冲突和额外开销。

**5. 动机/Insight**
   推理增强并不需要“越多知识越好”，关键在于找到**最小充分**的知识子集。真正有效的不是堆提示，而是筛出能支撑推理的紧凑 guidance。

**6. 解决方案**
   KnowRL 把 hint 设计转化为最小充分知识搜索问题：先将 guidance 分解为 atomic knowledge points，再通过约束子集搜索挑选紧凑且交互感知的知识子集。这样既缓解 reward sparsity，又减少无效信息干扰。

## 3-Rethinking OPD-Rethinking On-Policy Distillation of Large Language Models: Phenomenology, Mechanism, and Recipe

**1. source link**
   [https://arxiv.org/abs/2604.13016](https://arxiv.org/abs/2604.13016)

**2. 单位**
   Tsinghua NLP Group

**3. Keywords**
   On-policy Distillation, Post-training, Teacher-Student Alignment, Reasoning Models

**4. 问题背景**
   On-policy distillation 已成为 reasoning model 后训练的重要手段，但其为何有效、何时失败、失败机理是什么，过去并不清楚。很多做法更多像经验 recipe，而非机制清晰的方法。

**5. 动机/Insight**
   OPD 的关键不只是 teacher 更强，而是 **teacher 与 student 的 thinking pattern 是否兼容**。成功蒸馏通常要求 teacher 在 student 既有分布之外提供真正的新能力，同时二者在高概率 token 上保持一致。

**6. 解决方案**
   论文系统分析 OPD 的现象学与机制，并总结出可操作 recipe。核心是在 teacher-student 对齐与能力供给之间找到平衡，从而让蒸馏从“玄学技巧”变成更可解释的训练范式。

## 8-SPPO-SPPO: Sequence-Level PPO for Long-Horizon Reasoning Tasks

**1. source link**
   [https://arxiv.org/abs/2604.08865](https://arxiv.org/abs/2604.08865)

**2. 单位**
   NLP Group in SUSTech

**3. Keywords**
   PPO, Long CoT, Sequence-level RL, Contextual Bandit

**4. 问题背景**
   标准 token-level PPO 用于长链式推理时，常遭遇 credit assignment 不稳定与 value model 内存开销过大问题。推理轨迹越长，这些问题越严重。

**5. 动机/Insight**
   长 CoT 推理里，逐 token 建模 PPO 可能并不是最自然的形式。把问题提升到**sequence level**，可绕开部分时序 credit assignment 的不稳定性。

**6. 解决方案**
   SPPO 将 long-horizon reasoning 重新表述为 contextual bandit 风格的 sequence-level 优化问题，并配套设计解耦 value function。这样能在保持 PPO 思想的同时，提升长推理任务中的训练稳定性和效率。

**2. 单位**

**3. Keywords**
   Computer-use Agents, Safety Evaluation, Benign Instructions, Contextual Risk

**4. 问题背景**
   现有 computer-use agent 安全评测主要盯着显式恶意输入，如 misuse 或 prompt injection。可现实中很多风险并非来自恶意指令，而是 benign instruction 在特定上下文中引发危险执行结果。

**5. 动机/Insight**

**6. 解决方案**
   论文专门研究 benign-instruction setting 下的 agent 漏洞，并显示即便是 safety-aligned 模型也会出现很高攻击成功率。它推动安全评测从“恶意提示防护”扩展到“上下文结果安全”。

   该工作提出 Spatial Competence Benchmark，用于系统测量 frontier models 的空间能力。一个重要观察是：模型准确率会在较早阶段饱和，说明空间推理仍是明显短板。

## 40-SpotSound-SpotSound: Enhancing Large Audio-Language Models with Fine-Grained Temporal Grounding

**1. source link**
   [https://arxiv.org/abs/2604.13023](https://arxiv.org/abs/2604.13023)

**2. 单位**
   · 6 authors（HF 页面未给出明确主单位）

**3. Keywords**
   Audio-Language Model, Temporal Grounding, Long-form Audio, Timestamp Prediction

**4. 问题背景**
   长音频理解不仅要知道“发生了什么”，还要知道“在什么时候发生”。现有 audio-language model 在稀疏事件、噪声背景和时间戳预测方面仍不够精细。

**5. 动机/Insight**
   音频理解里的时间定位误差往往来自 false timestamps 和事件稀疏性。若 temporal grounding 做不好，再强的语义理解也难以真正服务长音频场景。

**6. 解决方案**
   SpotSound 通过更细粒度的 temporal grounding 机制提升长音频理解，重点抑制错误时间戳并增强对稀疏事件的定位能力。它面向的是更可用的 long-form audio understanding。

## 1. 4 月 15 日的热点很明显偏向 agent

GUI agent、long-horizon agent、multimodal search agent、多主体策略 agent 都很多，说明社区焦点已从“模型会不会答”进一步转向“模型能不能长期执行任务”。([Hugging Face][1])
