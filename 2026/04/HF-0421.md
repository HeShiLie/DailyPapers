## 1-`2604.18168`-`Extending One-Step Image Generation from Class Labels to Text via Discriminative Text Representation`

1. source link: https://arxiv.org/abs/2604.18168
2. 单位：AMAP-ML
3. Keywords：one-step generation, text-to-image, MeanFlow, discriminative text representation
4. 问题背景：One-step / few-step 图像生成最近很热，但 MeanFlow 类方法主要还停留在 class-to-image。把条件从离散类标签扩展到自由文本后，模型对文本表征的判别性要求会显著提高。
5. 动机/Insight：作者认为问题不只是“加个更强的 text encoder”这么简单，而是一步生成时，文本特征必须足够可分，才能在极少 refinement 中真正起作用。class label 效果好，本质上也来自其表征天然更离散、更可区分。
6. 解决方案：方法上引入更适合 one-step generation 的 LLM-based text encoder，并围绕判别性表征重新适配 MeanFlow 框架。最终把 one-step generation 从 class 条件扩展到了 text 条件，同时也在 diffusion setting 下验证了收益。

## 2-`2604.18486`-`OneVL: One-Step Latent Reasoning and Planning with Vision-Language Explanation`

1. source link: https://arxiv.org/abs/2604.18486
2. 单位：Xiaomi Research
3. Keywords：latent CoT, autonomous driving, world model, vision-language-action
4. 问题背景：自动驾驶里的 CoT reasoning 有助于规划和轨迹预测，但显式 token-by-token reasoning 延迟高，不适合实时部署。latent CoT 虽快，但通常又不如显式 CoT。
5. 动机/Insight：作者认为单纯“语言 latent”压缩掉了真正关键的因果世界动态，因此 latent reasoning 之所以弱，不是压缩本身不行，而是压错了对象。若让 latent 同时受语言解释和视觉世界模型监督，它就更可能学到可规划的因果结构。
6. 解决方案：OneVL 用统一 VLA + world model 框架，在 latent token 上加两个辅助解码器：一个重建文本 CoT，一个预测未来视觉 token。训练时分三阶段联合对齐，推理时去掉辅助头，一次并行填完 latent tokens，以 answer-only latency 获得更强性能。

## 7-`2604.14258`-`GFT: From Imitation to Reward Fine-Tuning with Unbiased Group Advantages and Dynamic Coefficient Rectification`

3. Keywords：post-training, group fine-tuning, imitation learning, reward fine-tuning
4. 问题背景：LLM post-training 里，SFT 擅长注入知识，RL 擅长优化目标，但两者如何平滑衔接一直不顺。直接靠单路径监督容易稀疏、偏置大，也会带来梯度不稳。

## 8-`2604.18574`-`When Can LLMs Learn to Reason with Weak Supervision?`

1. source link: https://arxiv.org/abs/2604.18574
2. 单位：论文页显示 6 authors，HF 首页未给单一机构
3. Keywords：weak supervision, reasoning, reward saturation, faithfulness
4. 问题背景：很多 reasoning 训练都想减少对显式 CoT 的依赖，转向弱监督或结果监督，但并不是所有情形都能学出可泛化推理。什么时候 weak supervision 真的够，是个基础问题。
5. 动机/Insight：作者指出能否学到 reasoning，不只取决于“有没有奖励”，还与 reward saturation 动态和 reasoning faithfulness 强相关。纯结果监督若过早饱和，模型很容易只学表面策略。
6. 解决方案：论文系统分析 weak supervision 下的泛化条件，并指出显式 trace 的 SFT 仍是关键支撑。结论更像一篇边界条件研究：告诉你什么时候弱监督行，什么时候最好别高估它。

## 12-`2604.16826`-`Crowded in B-Space: Calibrating Shared Directions for LoRA Merging`

1. source link: https://arxiv.org/abs/2604.16826
2. 单位：论文页显示 2 authors，HF 首页未给单一机构
3. Keywords：LoRA merging, model merging, interference, calibration
4. 问题背景：LoRA merge 很容易出现任务间互相干扰，尤其多个 adapter 合并时共享方向会挤在一起。简单线性 merge 往往伤害多任务性能。
5. 动机/Insight：作者观察到干扰并不是均匀分布的，而是集中在输出侧矩阵 B 的某些“拥挤方向”上。与其全局动刀，不如优先校准最容易冲突的那部分。
6. 解决方案：方法上单独校准 B-space 的共享方向，在尽量保留 task-specific 信息的同时缓解 merge interference。这个思路很轻量，也兼容现有 merging pipeline。

## 13-`2604.16830`-`The Illusion of Certainty: Decoupling Capability and Calibration in On-Policy Distillation`

1. source link: https://arxiv.org/abs/2604.16830
2. 单位：Salesforce AI Research
3. Keywords：on-policy distillation, calibration, confidence, deployment mismatch
4. 问题背景：On-policy distillation 常看能力提升，但模型置信度是否可靠却常被忽略。训练时上下文和部署时上下文不一致，也会让 calibration 进一步恶化。
5. 动机/Insight：作者强调 capability 和 calibration 不能混为一谈，模型变强不代表更可信。蒸馏后产生“确定性幻觉”，本质是训练-部署信息不匹配下的置信度偏移。
6. 解决方案：论文提出 calibration-aware 的分析与改进框架，在提升任务表现的同时修正 confidence reliability。重点不是单纯提分，而是让蒸馏后的输出“既对也别太装”。

6. 解决方案：论文从 generalizability、stability 和 embedding geometry 三方面系统分析开源 dense retriever。贡献主要在 diagnostic study 和 robustness insight。

## 46-`2604.15760`-`KWBench: Measuring Unprompted Problem Recognition in Knowledge Work`

1. source link: https://arxiv.org/abs/2604.15760
2. 单位：Clio AI
3. Keywords：knowledge work, problem recognition, benchmark, game-theoretic structure
4. 问题背景：很多知识工作 benchmark 默认问题定义已经给定，模型只需“按规格完成任务”。但在真实工作中，先识别出到底是什么问题，往往比解题本身更重要。
5. 动机/Insight：作者把 benchmark 往前推了一步：测模型能否在没有 prompt 指明结构时，从原始材料中识别 governing structure。这个“先认问题”能力长期被忽略。
6. 解决方案：KWBench 构建了 unprompted problem recognition benchmark，强调模型是否能从 raw inputs 中识别底层专业场景和博弈结构，而不是只做 extraction 或 completion。

## 47-`2604.15579`-`Symbolic Guardrails for Domain-Specific Agents: Stronger Safety and Security Guarantees Without Sacrificing Utility`

1. source link: https://arxiv.org/abs/2604.15579
2. 单位：Carnegie Mellon University
3. Keywords：agent safety, security guarantees, symbolic guardrails, high-stakes domains
4. 问题背景：高风险业务场景里的 agent 一旦误用工具，会造成隐私、财务等不可接受损失。训练式或 neural guardrail 常能减风险，但很难给出真正 guarantee。
5. 动机/Insight：作者认为高 stakes 环境不能只满足“平均更安全”，而要追求可验证的强约束。symbolic guardrails 因此比纯神经式保护更有现实意义。
6. 解决方案：论文研究面向 domain-specific agents 的 symbolic guardrails，通过显式策略约束提供更强 safety/security guarantee，同时尽量不牺牲 utility。

## 48-`2602.15143`-`Protecting Language Models Against Unauthorized Distillation through Trace Rewriting`

1. source link: https://arxiv.org/abs/2602.15143
2. 单位：论文页显示 4 authors
3. Keywords：distillation defense, trace rewriting, watermarking, reasoning traces
4. 问题背景：教师模型输出的 reasoning trace 很容易被拿去做未授权蒸馏，侵蚀 frontier model 的训练成本优势。如何既保留答案质量，又保护 trace 价值，是个现实问题。
5. 动机/Insight：作者认为防护点不一定在 answer，而可以放在 teacher trace 本身。若重写 reasoning trace，让其保留正确性但降低可蒸馏性，就能抬高盗蒸成本。
6. 解决方案：论文提出 trace rewriting 技术，在维持答案正确的同时，削弱学生模型从 trace 中提取能力的效果，并支持可检测 watermark。属于面向模型资产保护的防御工作。

## 49-`2604.03337`-`Significance and Stability Analysis of Gene-Environment Interaction using RGxEStat`

1. source link: https://arxiv.org/abs/2604.03337
2. 单位：Henan University
3. Keywords：gene-environment interaction, stability analysis, significance testing, statistics
4. 问题背景：G×E interaction 会显著影响性状在不同环境下的可预测性，因此需要同时分析显著性与稳定性。传统分析未必能统一兼顾这两点。
5. 动机/Insight：作者聚焦于更稳定、更系统地分析基因-环境交互，而不只是做单一显著性检验。对育种和表型预测场景都较关键。
6. 解决方案：论文提出 RGxEStat 用于 gene-environment interaction 的显著性与稳定性分析。整体偏统计方法与应用工具导向。
